# User-Level Token Usage Tracking
**Priority: HIGH — Billing & tier management foundation**

## Goal
Track how many AI tokens each individual user consumes, broken down by action and date. Show this inside the admin Users tab. Use token consumption as a proxy for usage intensity — the foundation for per-tier usage limits, upgrade prompts, and future metered billing.

---

## Where It Lives

### Admin Portal — Users Tab
Inside the existing user detail panel (the slide-in drawer on the right), add a new **Token History** section below the Profile Fields.

Shows:
- Total tokens consumed (lifetime)
- Total estimated cost generated by this user
- Usage history table: Date | Action | Tokens Used | Est. Cost
- Tier limit progress bar (once limits are configured)

### Admin Portal — Overview KPIs
Add one new KPI card: **Avg Tokens / User** — total tokens ÷ active users. Tracks platform efficiency.

---

## Token Attribution

Every AI call that already logs to `token_usage` needs a `user_id` column added. Current table logs operation + route + tokens but not which user triggered it.

### Actions to attribute per user:
| Action | Operation Label |
|---|---|
| Generate training plan | `plan_generation` |
| Generate For You feed card | `feed_generation` |
| Generate custom injury recovery | `injury_recovery_generation` |
| Generate exercise library details | `exercise_backfill` (admin only) |
| Coach AI generate program | `coach_program_generation` |
| Launchpad AI generate | `launchpad_generate` |
| CEO briefing / AMA | `ceo_brief`, `ceo_ask` |
| Knowledge base search | `knowledge_search` |

---

## Schema Change

```sql
-- Add user_id to existing token_usage table
ALTER TABLE token_usage
  ADD COLUMN IF NOT EXISTS user_id uuid REFERENCES auth.users(id) ON DELETE SET NULL;

-- Index for fast per-user lookups
CREATE INDEX IF NOT EXISTS token_usage_user_id_idx ON token_usage (user_id);
```

All existing API routes that insert to `token_usage` need to be updated to pass the authenticated user's ID.

## Per-Tier Usage Limits (Future Billing Layer)

Define soft limits per tier — once a user approaches their limit, the system prompts an upgrade. Once exceeded, AI features are gated.

| Tier | Monthly Token Budget | Action at Limit |
|---|---|---|
| Free | 50,000 tokens | Prompt to upgrade — AI features paused |
| F&F Beta | 500,000 tokens | Warning at 80% — no hard gate |
| Beta | 300,000 tokens | Warning at 80% — soft gate |
| Pro (future) | 1,000,000 tokens | Warning at 90% |
| Coach (future) | 2,000,000 tokens | Warning at 90% — covers client generation |

These limits are configurable in admin — no hardcoding.

---

## Admin Config Panel (Phase 2)

A simple settings section in the admin portal to configure:
- Token budget per tier
- Warning threshold (e.g., 80%)
- Hard gate vs. soft warning behavior
- Override: flag specific users as "unlimited" (for F&F, testers)

---

## User-Facing Limit Warning

When a user hits 80% of their tier's monthly budget:
- In-app banner: "You've used most of your AI credits this month. Upgrade to keep generating plans without limits."
- Links to upgrade flow (once Stripe is live)

When limit is reached:
- AI generation buttons disabled
- Message: "You've reached your monthly AI limit. Upgrade to continue."
