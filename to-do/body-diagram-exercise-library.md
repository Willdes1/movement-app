# Interactive Body Diagram + Exercise Library Search

---

## 10. Interactive Body Diagram / Muscle Selection Tool

### Goal
An interactive human body graphic where users tap a muscle group, joint, or body region and are shown relevant exercises, recovery movements, or targeted workouts.

### Current State
A "Jarvis-style" anatomical graphic exists somewhere in the app. It needs to be replaced with an accurate, realistic human body diagram with proper muscle, joint, and bone labeling.

### Target UX
1. User opens the body diagram screen
2. Sees front and back view of a realistic human body (toggle between views)
3. Taps a muscle group, joint, or region (e.g. "hamstrings", "left knee", "rotator cuff")
4. A panel slides up showing:
   - Exercises that target this area
   - Recovery/mobility movements for this area
   - Any injury/restriction flags the user has set (warns if they've logged this as injured)

### Body Regions (initial list)
**Upper body:** Neck, Shoulders (anterior/posterior deltoid), Rotator Cuff, Chest (upper/lower pec), Biceps, Triceps, Forearms, Upper Back (traps/rhomboids), Lats
**Core:** Abs (upper/lower rectus), Obliques, Spinal Erectors, Hip Flexors
**Lower body:** Glutes (maximus/medius), Hamstrings, Quads, Adductors, IT Band, Calves (gastroc/soleus), Tibialis Anterior
**Joints:** Shoulders, Elbows, Wrists, Hips, Knees, Ankles

### Graphic Approach (choose one)

| Option | Pros | Cons |
|---|---|---|
| SVG custom-built | Fully interactive, clickable regions, styled to match app | Requires design work to build accurate SVG |
| Licensed anatomy SVG | Professionally accurate | Licensing cost, may need modification |
| 3D model (Three.js) | Premium feel, rotatable | Significant dev complexity |
| Image map (HTML) | Simple, no SVG needed | Less flexible |

**Recommended Phase 1:** Custom SVG — front and back views, clickable regions mapped to muscle groups. Use a clean, dark-mode aesthetic that fits the app. Not photo-realistic, but anatomically accurate schematic.

### Where It Lives
- `/browse` → "Body Map" tab (alongside exercise search)
- Or a dedicated `/body-map` page linked from nav

### Data Connection
Each muscle region maps to tags on the exercise_library table. Exercises already have targeting info — confirm the schema has a `muscle_groups` or `tags` column, or add one.

## 11. Exercise Library Search & Targeted Movement Area

### Goal
A searchable, browsable exercise library where users can:
- Search by exercise name ("hammer curl", "deadlift")
- Filter by muscle group (from body diagram or dropdown)
- Filter by equipment needed (bodyweight, barbell, dumbbell, resistance band, etc.)
- Filter by movement type (strength, mobility, cardio, corrective, cooldown)
- View exercise details: how to perform it, muscles targeted, coaching cues
- See YouTube link / video when available

### Current State
`exercise_library` table exists with coaching details, cues, and technique notes generated by AI. `exercise_media` table exists with YouTube/image links. Auto-sync trigger between them exists.

### Features

**Search:**
- Real-time search-as-you-type by exercise name
- Fuzzy matching (handles "bench" → Bench Press, Incline Bench, etc.)

**Filters:**
- Muscle group (multi-select or body diagram selection)
- Equipment (multi-select)
- Movement type / category
- Difficulty level

**Exercise Card:**
- Name, primary muscles, equipment needed
- How-to instructions (from exercise_library.coaching_cues or similar)
- YouTube embed or link if available (from exercise_media)
- "Add to my session" button (future — lets users build custom sessions)

**Cooldown / Recovery Use Case:**
User finishes lower body workout, hips feel tight → taps "Hips" on body diagram → sees hip flexor stretches, pigeon pose, 90/90 hip stretch → taps "Add to tonight's cooldown"

**Arm pump use case:**
User wants to add arm work → searches "bicep" → sees hammer curls, barbell curls, preacher curls → taps exercise → sees full coaching detail + video

### Schema Additions (if not already present)
```sql
ALTER TABLE exercise_library ADD COLUMN IF NOT EXISTS muscle_groups text[];   -- e.g. ['hamstrings', 'glutes', 'lower_back']
ALTER TABLE exercise_library ADD COLUMN IF NOT EXISTS equipment     text[];   -- e.g. ['barbell', 'rack']
ALTER TABLE exercise_library ADD COLUMN IF NOT EXISTS category      text;     -- 'strength', 'mobility', 'cardio', 'corrective', 'cooldown'
ALTER TABLE exercise_library ADD COLUMN IF NOT EXISTS difficulty    text;     -- 'beginner', 'intermediate', 'advanced'
```

### Build Phases

**Phase 1 — Exercise Search**
1. `/browse` → Exercise search with name search + basic filters
2. Exercise card with coaching detail
3. YouTube link display (from exercise_media)

**Phase 2 — Body Diagram**
4. SVG body diagram (front + back, clickable regions)
5. Body region → exercise filter connection
6. Injury awareness (warn if selected region matches user's restriction_areas)

**Phase 3 — Custom Session Builder**
7. "Add to session" from any exercise
8. User-built custom cooldown / workout sessions (saved to DB)
9. These sessions can be scheduled or used ad-hoc
